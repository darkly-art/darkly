## Modularity Principle

Module-specific code lives in the module, and nowhere else. No central match statements, no central registries with per-module fields, no dispatch tables that must be edited when a module is added. Each filter is a self-contained file that implements the `Filter` trait and exports a `register()` function — adding a new filter means creating one `.rs` file in the `filters/` directory. A `build.rs` script scans the directory and generates the module declarations and registry. No other file needs to be touched.

## Step 7: Filter shader system

### `gpu/filter.rs` — unified trait and infrastructure

```rust
/// Shared GPU resources for a filter type (pipeline + bind group layout).
/// Arc-wrapped so multiple filter instances of the same type share them.
pub struct FilterPipeline {
    pub pipeline: wgpu::RenderPipeline,
    pub bind_group_layout: wgpu::BindGroupLayout,
}

/// Unified trait for filters. Each filter is one struct that holds both its
/// user-facing parameters and an Arc to the shared GPU pipeline.
pub trait Filter: std::fmt::Debug {
    fn type_id(&self) -> &'static str;
    fn clone_boxed(&self) -> Box<dyn Filter>;
    fn pass_count(&self) -> u32;
    fn pipeline(&self) -> &wgpu::RenderPipeline;
    fn bind_group_layout(&self) -> &wgpu::BindGroupLayout;
    fn create_cache(
        &self, device: &wgpu::Device, queue: &wgpu::Queue,
        accum_views: &[wgpu::TextureView; 2], sampler: &wgpu::Sampler,
        canvas_width: u32, canvas_height: u32,
    ) -> FilterLayerCache;
}

/// Cached GPU objects for a filter layer instance (P1).
pub struct FilterLayerCache {
    pub uniform_bufs: Vec<wgpu::Buffer>,
    pub bind_groups: Vec<[wgpu::BindGroup; 2]>,
    pub aux_textures: Vec<wgpu::Texture>,
    pub aux_views: Vec<wgpu::TextureView>,
}

/// What each filter module returns from its `register()` function.
pub struct FilterRegistration {
    pub type_id: &'static str,
    pub create_pipeline: fn(&wgpu::Device, wgpu::TextureFormat) -> FilterPipeline,
    pub from_js: fn(JsValue, Arc<FilterPipeline>) -> Box<dyn Filter>,  // wasm32 only
}

/// Auto-discovered filter registry with lazy pipeline caching.
pub struct FilterRegistry {
    entries: HashMap<&'static str, RegistryEntry>,
}

impl FilterRegistry {
    pub fn new() -> Self;              // populates from filters::registrations()
    pub fn pipeline(&mut self, type_id, device, format) -> Arc<FilterPipeline>;
    pub fn create_filter(&mut self, type_id, js, device, format) -> Box<dyn Filter>;
}
```

### Noise texture generation (inside `Noise::create_cache`)

1. Compute noise texture dimensions: `ceil(canvas_width / resolution) x ceil(canvas_height / resolution)`
2. Fill with random `u8` values (one channel — luminance noise) using simple xorshift PRNG
3. Upload to `R8Unorm` GPU texture via `queue.write_texture()`
4. Store in `FilterLayerCache::aux_textures`

The noise texture is static — generated once at layer creation, does not change per-frame.

### Noise shader (`shaders/filters/noise.wgsl`)

```wgsl
struct NoiseParams { amount: f32, resolution: f32, _pad0: f32, _pad1: f32 }
@group(0) @binding(0) var t_input: texture_2d<f32>;    // accumulator (composite so far)
@group(0) @binding(1) var t_noise: texture_2d<f32>;    // pre-generated noise texture (R8)
@group(0) @binding(2) var t_sampler: sampler;
@group(0) @binding(3) var<uniform> params: NoiseParams;

@fragment fn fs_noise(@builtin(position) pos: vec4f) -> @location(0) vec4f {
    let dims = vec2f(textureDimensions(t_input));
    let uv = pos.xy / dims;
    let color = textureSample(t_input, t_sampler, uv);

    let noise_val = textureSample(t_noise, t_sampler, uv).r;

    // Overlay blend mode
    let noise_rgb = vec3f(noise_val);
    let base = color.rgb;
    let lo = 2.0 * base * noise_rgb;
    let hi = 1.0 - 2.0 * (1.0 - base) * (1.0 - noise_rgb);
    let blended = select(hi, lo, base < vec3f(0.5));

    return vec4f(mix(color.rgb, blended, params.amount), color.a);
}
```

### Verification

Add noise filter between two raster layers. Bottom layer has gradient, top is painted. Noise filter applies visible grain overlay.

### `frontend/src/App.svelte`

- On mount: call `initEditor(canvas)` from `editor.ts`
- `mousedown` -> `handle.snapshot(paintLayerId)` then `handle.paint(...)`
- `mousemove` with button down -> `handle.paint(...)`
- `mouseup` -> `handle.commit()`
- `keydown` Ctrl+Z -> `handle.undo()`, Ctrl+Shift+Z -> `handle.redo()`
- `requestAnimationFrame` loop -> `handle.render()`
- On mount after init, hardcode demo setup:
  ```ts
  const bg = handle.add_raster_layer();
  handle.fill_gradient(bg);
  const noise = handle.add_filter_layer("noise", { amount: 0.5, resolution: 2 });
  const paint = handle.add_raster_layer();
  // mouse painting targets `paint` layer
  ```
