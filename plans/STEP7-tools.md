## Architecture

The tool system follows the exact same auto-discovery pattern as filters (see AGENTS.md modularity principle):

| Filter system | Tool system | Role |
|--------------|-------------|------|
| `gpu/filter.rs` — `Filter` trait | `tool.rs` — `ToolOp` trait | Trait definition |
| `gpu/filters/mod.rs` — @generated | `tools/mod.rs` — @generated | Auto-discovered modules |
| `gpu/filters/noise.rs` — self-contained | `tools/paint_circle.rs` — self-contained | One file per module |

**The split between Rust and TypeScript:**

| Concern | Where |
|---------|-------|
| Gesture interpretation (what does the input mean?) | TypeScript `Tool` |
| Coordinate transform (screen -> canvas) | TypeScript (calls `handle.screen_to_canvas()`) |
| Tool switching, hotkeys, UI options panels | TypeScript |
| Stroke lifecycle (begin/end transaction) | Rust `DarklyHandle` |
| Pixel modification (painting, filling, gradient) | Rust `Document` |
| Dirty tracking | Rust `Document` |
| Undo integration | Rust (transaction at stroke boundary) |

## Step 5a: Rust side — tool operation modules

### `crates/darkly/src/tool.rs` — Tool operation trait + registry

```rust
use crate::document::Document;
use crate::layer::LayerId;
use std::collections::HashMap;
#[cfg(target_arch = "wasm32")]
use wasm_bindgen::JsValue;

pub struct ToolRegistration {
    pub type_id: &'static str,
    #[cfg(target_arch = "wasm32")]
    pub from_js: fn(JsValue) -> Box<dyn ToolOp>,
}

pub trait ToolOp {
    fn apply(&self, doc: &mut Document, layer_id: LayerId);
}

pub struct ToolRegistry {
    entries: HashMap<&'static str, RegistryEntry>,
}

struct RegistryEntry {
    #[cfg(target_arch = "wasm32")]
    from_js: fn(JsValue) -> Box<dyn ToolOp>,
}

impl ToolRegistry {
    pub fn new() -> Self; // populates from tools::registrations()

    #[cfg(target_arch = "wasm32")]
    pub fn apply(&self, type_id: &str, js: JsValue,
                 doc: &mut Document, layer_id: LayerId);
}
```

### `crates/darkly/src/tools/paint_circle.rs`

Fully self-contained. Iterates tiles touched by circle's bounding box, writes pixels within radius, marks dirty. This is the same logic currently in `Document::paint_circle()` — extract it into a tool op.

```rust
pub fn register() -> ToolRegistration {
    ToolRegistration {
        type_id: "paint_circle",
        #[cfg(target_arch = "wasm32")]
        from_js: |js| {
            let p: PaintCircleParams = serde_wasm_bindgen::from_value(js).unwrap();
            Box::new(PaintCircleOp { x: p.x, y: p.y, radius: p.radius, color: [p.r, p.g, p.b, p.a] })
        },
    }
}

pub struct PaintCircleOp {
    pub x: f32, pub y: f32, pub radius: f32, pub color: [u8; 4],
}

impl ToolOp for PaintCircleOp {
    fn apply(&self, doc: &mut Document, layer_id: LayerId) {
        // iterate tiles, write pixels within radius, mark dirty
    }
}
```

## Step 5b: WASM bridge — stroke lifecycle

The WASM bridge exposes a three-method stroke lifecycle that maps to undo transactions:

```rust
impl DarklyHandle {
    /// Begin a stroke on a layer. Starts an undo transaction.
    pub fn begin_stroke(&mut self, layer_id: u64) {
        self.doc.begin_transaction(layer_id);
        self.active_stroke_layer = Some(layer_id);
    }

    /// Apply a stroke operation. Delegates to ToolRegistry.
    pub fn stroke_to(&mut self, op_type: &str, params: &JsValue) {
        let layer_id = match self.active_stroke_layer {
            Some(id) => id,
            None => return,
        };
        self.tool_registry.apply(op_type, params.clone(), &mut self.doc, layer_id);
    }

    /// End the current stroke. Commits the undo transaction.
    pub fn end_stroke(&mut self) {
        if let Some(layer_id) = self.active_stroke_layer.take() {
            if let Some(step) = self.doc.commit_transaction(layer_id) {
                self.undo_stack.push(step);
            }
        }
    }
}
```

Key insight from Krita: **the tool never manages undo.** `begin_stroke` opens a transaction, `end_stroke` commits it. The TS tool doesn't know about transactions.

Also add `pick_color` for the color picker tool:
```rust
pub fn pick_color(&self, x: f32, y: f32) -> Vec<u8>;
```

This reads from the composite cache via GPU readback (or from tile data directly for Phase 2).

## Step 5c: TypeScript side — gesture interpreters

### `frontend/src/tools/registry.ts`

```typescript
export interface ToolContext {
    handle: DarklyHandle;
    screenToCanvas: (screenX: number, screenY: number) => { x: number; y: number };
}

export interface Tool {
    readonly id: string;
    readonly name: string;
    readonly icon: string;
    readonly hotkeyAction: string;
    readonly optionsComponent?: Component;

    onActivate?(ctx: ToolContext): void;
    onDeactivate?(ctx: ToolContext): void;
    onPointerDown(ctx: ToolContext, e: PointerEvent, canvasX: number, canvasY: number): void;
    onPointerMove(ctx: ToolContext, e: PointerEvent, canvasX: number, canvasY: number): void;
    onPointerUp(ctx: ToolContext, e: PointerEvent): void;
}

class ToolRegistry {
    private tools = new Map<string, Tool>();
    private order: string[] = [];

    register(tool: Tool) { ... }
    get(id: string): Tool | undefined { ... }
    all(): Tool[] { ... }
}

export const toolRegistry = new ToolRegistry();
```

### `frontend/src/tools/brush.svelte.ts`

Exports `BRUSH_HOTKEY = 'KeyB'`. Reads `app.foreground`, `app.brushSize`, `app.brushOpacity` at each pointer event (mid-stroke parameter changes work). `onPointerDown` calls `begin_stroke(BigInt(layerId))` then `stroke_to('paint_circle', ...)`. `onPointerMove` calls `stroke_to`. `onPointerUp` calls `end_stroke`.

### `frontend/src/tools/eraser.svelte.ts`

Exports `ERASER_HOTKEY = 'KeyE'`. Shares `app.brushSize` / `app.brushOpacity` with brush (Krita behavior). Calls `stroke_to('erase_circle', ...)`.

### `frontend/src/tools/fill.svelte.ts`

Exports `FILL_HOTKEY = 'KeyF'`. One-shot tool: `onPointerDown` does `begin_stroke` -> `stroke_to('flood_fill', ...)` -> `end_stroke`.

### `frontend/src/tools/gradient.svelte.ts`

Exports `GRADIENT_HOTKEY = 'KeyG'`. Two-point tool: `onPointerDown` records start, `onPointerUp` calls `stroke_to('linear_gradient', ...)` -> `end_stroke`.

### `frontend/src/tools/colorpicker.svelte.ts`

Exports `COLORPICKER_HOTKEY = 'KeyP'`. No stroke — read-only. `onPointerDown` calls `handle.pick_color(cx, cy)` and sets `app.foreground`.
